  - SQLite

- ✅ **数据库连接管理**

## 功能演示

### 1. SQL 查询与 AI 智能生成

支持传统 SQL 编辑器和 AI 自然语言查询，AI 会自动分析表结构并生成准确的 SQL 语句。

![SQL查询界面](docs/images/查询.png)

**主要功能**：
- 🔹 SQL 语法高亮编辑器
- 🔹 AI 自然语言转 SQL（支持中英文）
- 🔹 多标签页结果展示
- 🔹 查询结果直接编辑（自动生成 UPDATE 语句）
- 🔹 数据导出（CSV、Excel、JSON）
- 🔹 结果表格支持排序、筛选

### 2. AI 模型配置

支持配置多个 AI 模型，兼容 OpenAI API 格式（支持本地大模型如 Ollama）。

![AI配置界面](docs/images/ai配置.png)

**配置项**：
- 🔹 模型名称和 API 配置
- 🔹 Token 使用统计
- 🔹 多模型切换
- 🔹 自定义 API 端点（支持本地模型）

### 3. AI 多轮对话创建表

通过自然语言对话快速创建数据库表，AI 自动识别当前数据库类型并生成符合规范的建表脚本。

![AI多轮对话创建表](docs/images/多轮对话新建表.png)

**智能特性**：
- 🔹 多轮对话式交互（支持逐步完善需求）
- 🔹 自动识别数据库类型（MySQL/PostgreSQL/SQLite等）
- 🔹 **智能识别现有表规范**：AI 会自动分析当前数据库中已有表的结构，学习命名规范、字段类型选择、主键设置等风格
- 🔹 **参考表结构风格**：创建新表时会选择最相关的现有表作为参考，保持整个数据库设计风格的一致性
- 🔹 智能字段类型推荐（基于参考表的字段类型选择习惯）
- 🔹 自动生成主键、索引和约束（遵循已有表的设置方式）
- 🔹 符合数据库规范的建表语句
- 🔹 一键执行创建表

### 4. 表结构编辑和管理

可视化编辑表结构，支持 AI 辅助快速创建表和修改字段。

![编辑表界面](docs/images/编辑表.png)

**编辑功能**：
- 🔹 字段管理（添加/删除/修改）
- 🔹 数据类型选择（自适应不同数据库）
- 🔹 索引管理
- 🔹 **AI 智能修改表结构**：基于当前表结构和自然语言描述，智能生成 ALTER TABLE 语句
- 🔹 **风格一致性**：修改表时保持与现有表结构风格一致的命名规范和类型选择
- 🔹 **信息可复制** 🆕：表名、字段信息等均可选择和复制，支持右键复制选中或全部字段
- 🔹 SQL 预览和执行

**表管理功能**：
- 🔹 右键点击表节点可执行多种操作
- 🔹 **删除表** 🆕：双重确认机制，安全删除不需要的表
- 🔹 复制表结构
- 🔹 在新标签页中查询表数据

### 5. 表结构同步

在多个数据库之间同步表结构，支持批量操作。

![表结构同步](docs/images/表结构同步.png)

**同步功能**：
- 🔹 源数据库和目标数据库选择
- 🔹 表结构差异对比
- 🔹 批量同步多个表
- 🔹 SQL 脚本生成
- 🔹 安全的变更预览

## 使用说明

### 1. 添加数据库连接

**方式一：手动添加**
1. 点击菜单 "文件" -> "添加数据库连接" 或工具栏的 "添加连接" 按钮
2. 填写连接信息：
   - 连接名称（自定义）
   - 数据库类型
   - 主机地址
   - 端口
   - 数据库名
   - 用户名
   - 密码
3. 点击 "确定" 保存连接

**方式二：从 Navicat 导入** 🆕
1. 点击菜单 "文件" -> "从 Navicat 导入" 或工具栏的 "导入 Navicat" 按钮
2. 在导入对话框中：
   - 点击 "自动检测 Navicat 连接" 自动从注册表或配置文件导入（Windows）
   - 或点击 "从 .ncx 文件导入" 选择 Navicat 导出的 `.ncx` 文件
3. 选择要导入的连接（支持多选）
4. 点击 "确定" 导入选中的连接

**如何从 Navicat 导出连接**:
- 在 Navicat 中，选择 "文件" -> "导出连接" 或 "工具" -> "导出连接"
- 选择要导出的连接，保存为 `.ncx` 文件
- 在本工具中选择该 `.ncx` 文件导入

**注意**: 
- Windows 系统会自动从注册表读取 Navicat 连接
- 支持导入 Navicat 导出的 `.ncx` 文件格式
- 某些版本的 Navicat 密码可能无法自动解密，需要手动输入密码

### 2. 执行 SQL 查询

1. 在连接树中选择一个数据库连接
2. 在 SQL 编辑器中输入 SQL 语句
3. 按 `F5` 或点击 "执行" 按钮执行查询
4. 查询结果会显示在下方的结果表格中

### 3. AI 智能 SQL 生成 🤖

1. 在 AI 输入框中输入自然语言描述
2. AI 会自动：
   - 选择相关表
   - 分析表结构
   - 生成准确的 SQL 语句
3. 生成的 SQL 会自动执行并显示结果

**特性**：
- **智能表选择**：如果 SQL 编辑器中已有脚本，当输入不限定表名时（如"查询name字段"、"添加条件"），AI 会优先使用编辑器中的表，而不是重新选择
- 数据库类型感知：SQL 语法会根据数据库类型自动适配（MySQL、PostgreSQL 等）
- 表结构分析：自动使用正确的表名和列名
- 枚举字段识别：自动识别并使用正确的枚举值

**使用技巧**：
- 💡 在查看某个表的数据后，可以直接输入"只看active的"、"按时间排序"等简短指令，AI 会基于当前表生成 SQL
- 💡 如果需要查询其他表，明确指定表名即可，如"查询 users 表"

### 4. 管理连接

- **编辑连接**: 右键点击连接 -> "编辑"
- **测试连接**: 右键点击连接 -> "测试连接"
- **删除连接**: 右键点击连接 -> "删除"

### 5. 语言设置 🌐

1. 点击菜单 "设置" -> "设置"
2. 选择您偏好的语言（中文/英文）
3. 重启应用使更改生效

**注意**: 语言设置会保存在 Windows 注册表（Windows）或配置文件（其他平台）中。

## 技术栈

- **GUI框架**: PyQt6
- **数据库**: SQLAlchemy
- **数据验证**: Pydantic
- **密码解密**: cryptography (用于 Navicat 导入)
- **AI集成**: OpenAI API 兼容
- **国际化**: 自定义 i18n 系统（支持 JSON/TS 文件）
- **Python版本**: >= 3.8

## 安装

1. 创建虚拟环境（推荐）:
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows
```

2. 安装依赖:
```bash
pip install -r requirements.txt
```

**注意**: 根据您使用的数据库类型，可能需要安装对应的驱动：
- MySQL: `pymysql` (已包含)
- PostgreSQL: `psycopg2-binary` (已包含)
- Oracle: `cx-Oracle` (可选)
- SQL Server: `pyodbc` (已包含)
